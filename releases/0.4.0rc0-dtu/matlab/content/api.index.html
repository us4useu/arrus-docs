API Reference¶

Caution

ARRUS is currently under development and its API will be modified in the future. Please expect breaking changes.

Us4R handle¶

class arrus.Us4R(nArius, probeName, voltage, logTime)¶

Bases: handle

A handle to the Us4R system.

This class provides functions to configure the system and perform data acquisition using the Us4R.

Please note: only one instance of this class can be in use at a time!

Us4R(nArius, probeName, voltage, logTime)¶

Us4R handle constructor.

Parameters

  • nArius – number of arius modules available in the us4R system

  • probeName – name of the probe to use

  • voltage – a voltage to set, should be in range 0-90 [Vpp]

  • logTime – set to true if you want to display acquisition and reconstruction time (optional)

Returns

Us4R instance

run()¶

Runs uploaded operations in the us4R system.

Currently, only supports SimpleTxRxSequence and Reconstruction implementations.

Returns

RF frame and reconstructed image (if Reconstruction operation was uploaded)

runLoop(isContinue, callback)¶

Runs the uploaded operations in a loop.

Currently, only supports SimpleTxRxSequence and Reconstruction implementations.

Parameters

  • isContinue – should the system continue executing the op? Takes no parameters and returns a boolean value.

  • callback – a function to call after executing the operation. Should take one parameter, which will be feed with the output of the executed op.

upload(sequenceOperation, reconstructOperation)¶

Uploads operations to the us4R system.

Currently, only supports SimpleTxRxSequence and Reconstruction implementations.

Parameters

  • sequenceOperation – TX/RX sequence to perform on the us4R system

  • reconstructOperation – reconstruction to perform with the collected data

Returns

updated Us4R object

Tx/Rx sequences¶

We define Tx/Rx sequence with a common parameters.

class arrus.SimpleTxRxSequence(varargin)¶

Bases: arrus.Operation

A sequence of Tx/Rx operations to perform on a device.

Parameters

  • txCenterElement – vector of center elements of tx aperture [element] (for LINSequence)

  • txAperturecenter – vector of tx aperture center positions [m]

  • txApertureSize – size of the tx aperture [element]

  • txFocus – tx focal length [m]

  • txAngle – tx angle [rad]

  • speedOfSound – speed of sound for [m/s]

  • txFrequency – tx frequency [Hz]

  • txNPeriods – number of sine periods in the tx burst (can be 0.5, 1, 1.5, etc.)

  • rxNSamples – number of recorded samples per channel [sample]

  • txPri – tx pulse repetition interval [s]

We have implemented specific Tx/Rx sequences:

class arrus.PWISequence(varargin)¶

Bases: arrus.SimpleTxRxSequence

Plane Wave Imaging Tx/Rx Sequence.

The length of the PWI sequence is determined by the length of txAngle. The txFocus and txApertureCenter must be scalars or be the same length as txAngle. The PWI sequence has txFocus set to inf by defualt.

class arrus.STASequence(varargin)¶

Bases: arrus.SimpleTxRxSequence

Synthetic Transmit Aperture Tx/Rx Sequence.

The length of the STA sequence is determined by the length of txApertureCenter. The txFocus and txAngle must be scalars or be the same length as txApertureCenter.
